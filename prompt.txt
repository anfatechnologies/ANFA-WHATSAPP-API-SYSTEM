Role: Senior Full-Stack Architect.
Objective: Build a robust, real-time Settings Module for the ANFA WhatsApp Platform that is fully synced between Backend (FastAPI/PostgreSQL) and Frontend (Next.js/React).

Task: Implement the following 4 Settings Categories (DB + UI + API):

1. Meta Business Configuration (API Settings):

Fields: WhatsApp_Business_Account_ID, Phone_Number_ID, Permanent_Access_Token, App_Secret.

Logic: These must be stored in the DB (encrypted) and loaded into the worker.py context dynamically so changes take effect without restarting containers.

2. Automation & Workflow Settings (Integration):

Fields: n8n_Webhook_URL, Auto_Reply_Enabled (Toggle), Default_Reply_Message.

Logic: Toggle these to enable/disable automated message handling in the background worker.

3. System & Privacy (Data Sovereignty):

Fields: Data_Retention_Days (Integer), Encryption_Key_Rotation (Button), Enable_Logging (Toggle).

Logic: If Data_Retention_Days is changed, trigger a background task to clean up old logs/messages in PostgreSQL.

4. Appearance & UI Preferences:

Fields: Theme_Mode (Dark/Light/System), Notification_Sound_Enabled (Toggle), Language_Selection.

Logic: Persist these in local storage but sync with DB for multi-device cross-session experience.

Technical Requirements:

Sync: Implement a GET /api/settings/live endpoint that streams updates to the frontend using the same SSE (Server-Sent Events) pipe we used for messages. This ensures settings changes are reflected instantly without page refresh.

Security: POST /api/settings/update must validate input, encrypt secrets, and log the admin change to the immutable audit trail.

Frontend: Create a reusable SettingsForm component using Shadcn/UI that binds to the React Query state.

Instruction for Agent:

Start by defining the Settings Pydantic model and SQLAlchemy table.

Create the unified SettingsService in the backend to handle caching and real-time broadcasting.

Generate the React component that subscribes to the setting changes via SSE.
